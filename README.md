# parking-violations
Дипломный проект для курса IBS Center. Java middle deveoper

Приложение, которое анализирует сведения о нарушениях правил парковки в г. Филадельфия (США), например, стоянка в неположенных местах, уклонение от оплаты за парковку, превышение лимита времени и прочее.

## Формат исходных данных
На вход подаются 4 файла (parking содержат одинаковую информацию, но в разных форматах):
```
parking.csv
parking.json
properties.csv
population.txt
```

Файлы содержат информацию о парковочных нарушениях, сведения о недвижимости и информацию о численности жителей по каждому ZIP-коду.

## Порядок следования аргументов командной строки:

+ формат файла с парковочными нарушениями: csv или json
+ имя файла с парковочными нарушениями
+ имя файла с данными о недвижимости
+ имя файла с данными о населении
+ имя файла-журнала (подробнее о журнале см. далее)
Пример команды:
```
java -jar diploma.jar json parking.json properties.csv population.txt log
```
При вводе ошибочных данных командной строки в консоль выводится сообщение, соответствующее ошибке.
## Решаемые задачи

При успешном запуске выводится меню со следующими пунктами:
0. выход (Exit)

1. общая численность населения (Total Population)

2. средняя сумма всех штрафов на душу населения (Total Parking Fines per Capita)

3. средняя рыночная стоимость объектов жилой недвижимости (Average Market Value)

4. среднее значение совокупной жилой площади (Average Total Livable Area)

5. общая стоимость объектов жилой недвижимости на душу населения (Total Market Value per Capita)

6. расчет произвольной характеристики (Additional Feature):
   
   Для каждого ZIP-кода возвращает пару значений:
+ Средняя рыночная стоимость недвижимости
+ Среднее число штрафов на жителя

## Журналирование
Помимо печати результатов в консоли, приложение фиксирует вводимые данные от пользователя; это делается в журнальном файле, чье имя задается последним аргументом командной строки. Если указанного файла еще не существует, он должен быть создан. Данные просто добавляются в файл, не перезаписывая его с самого начала.

При всяком запуске приложения в журнал надо записывается таймстамп текущего времени в миллисекундах, затем перечисляются аргументы командной строки; все указанные поля такой записи разделяются одиночными пробелами.

Всякий раз, когда открывается очередной файл для считывания данных, в журнале отмечается таймстамп этого события в миллисекундах и имя файла.

Когда пользователь вводит ту или иную команду в ответ на приглашение, это также должно отмечаться в журнале, т.е. указывается таймстамп события в миллисекундах и введенная цифра выбранной команды.
